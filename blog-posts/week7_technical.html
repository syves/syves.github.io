SQL Injections are Bad

why and how to stop injections

Why?!

  1. OH noes! SOME people like to insert destructive statements into input fields to break tables!

  2. It is one of the top 10 security issues

    HOW do they do it?

    1. Classic sequel injection

  1. 1. Incorrectly filtered strings

  2. -that include equality tests, and SQL actions

  3. 2. Incorrect use of type or no filtering of type

  4. -allows users to pass in a SQL command

    5. 2. Blind injection

  5. the attacker inserts a logical statement in order to test for vulnerability

    8. 3. Data base management system specific SQLI

    9. 4. Compound

  9. Injection and other nasty things!

What can I do? BEAR spray!!

  1. 1. Escaping

  2. -escape characters that have a special meaning in sequel. ex 'DUMP'

  3. 2. Parameterized statements

  4. -use bind variables, input will not be evaluated immediately and will be passed to another check to be escaped

  5. 3. Pattern check

  6. -Finally! enforcement of type with checking for pattern recognition

  7. ex. is 'DUMP table' an int?

  8. 4. Database permissions

  9. -Limiting the permissions on the database login!
